# NANO BANANA 2 — MASTER UX/UI SKILL

This workspace follows the Nano Banana 2 design and coding methodology.

All agents must obey the following UX/UI principles.

---

## CORE DESIGN PHILOSOPHY

1. Interfaces must prioritize **clarity, speed, and simplicity**.
2. UI components must be **modular and reusable**.
3. Design must follow a **consistent design system** across all pages.

---

## LAYOUT SYSTEM

• Use an **8px grid spacing system**
• Use **max-width containers**
• Use **flexbox or grid layouts**
• Maintain consistent vertical rhythm

Example spacing:

8px
16px
24px
32px
48px
64px

---

## TYPOGRAPHY

Primary Font: Inter

Hierarchy:

H1 — 48px bold  
H2 — 36px bold  
H3 — 28px semibold  
Body — 16px regular  
Small — 14px

Line height: 1.4–1.6

---

## COLOR SYSTEM

Use a minimal color palette:

Primary
Secondary
Background
Surface
Accent
Error

All colors must meet **WCAG accessibility contrast standards**.

---

## COMPONENT RULES

All UI elements must be reusable components:

Buttons
Cards
Navigation
Forms
Modals
Tables

Component naming format:

```text
component-name.tsx
component-name.css
```

---

## RESPONSIVE DESIGN

All layouts must support:

Desktop  
Tablet  
Mobile

Breakpoints:

640px  
768px  
1024px  
1280px

---

## UX PRINCIPLES

Agents must prioritize:

• minimal cognitive load  
• intuitive navigation  
• fast loading interfaces  
• accessible design  

Never produce cluttered interfaces.

---

## CODING STANDARDS

Preferred stack:

React  
TailwindCSS  
TypeScript

Rules:

• Keep components under 200 lines
• Separate logic from UI
• Use semantic HTML
• Optimize for performance
